# Workspace Folder Stash（工作区文件夹暂存）

一个为 VS Code 多根工作区带来 Eclipse 风格的关闭/重新打开功能的扩展。在不移除文件夹的情况下临时隐藏根文件夹。

## 功能特性

- **临时关闭文件夹** — 在 Explorer 侧边栏中隐藏选中的根文件夹，同时将其保留在工作区配置中
- **重新打开文件夹** — 从可搜索的列表中恢复之前关闭的文件夹
- **永久移除** — 通过重新打开选择器中的垃圾桶按钮从工作区配置中彻底删除暂存的文件夹
- **多选支持** — 一次关闭或重新打开多个文件夹
- **非破坏性操作** — 所有操作仅编辑 `.code-workspace` 文件；不会丢失任何数据

## 使用方法

### 临时关闭文件夹

1. 在 Explorer 侧边栏中右键点击一个根文件夹
2. 选择 **"Temporarily Close Folder"**（临时关闭文件夹）
3. 该文件夹从 Explorer 中消失，但仍保留在工作区文件的 `stashedFolders` 中

### 重新打开文件夹

1. 在 Explorer 中的任意位置右键点击
2. 选择 **"Reopen Folder"**（重新打开文件夹）
3. 从快速选择列表中选择一个或多个暂存的文件夹
4. 按 Enter 将它们恢复到 Explorer

### 永久删除文件夹

1. 打开"重新打开文件夹"选择器（见上方）
2. 点击任何暂存文件夹旁的垃圾桶图标（🗑️）
3. 该文件夹被永久从工作区配置中删除

## 工作原理

本扩展通过以下方式修改活跃的 `.code-workspace` 文件：
- 将关闭的文件夹从 `folders[]` 数组移至新的 `stashedFolders[]` 数组
- 当你重新打开文件夹时，将其恢复到 `folders[]`
- 使用垃圾桶按钮时，永久删除文件夹条目

所有更改都直接保存到工作区文件中，你可以在需要时手动查看和编辑它们。

## 系统要求

- VS Code 1.74.0 或更高版本
- 多根工作区（`.code-workspace` 文件）

## 扩展设置

无需额外配置。安装后即可立即使用。

## 已知限制

- 仅适用于多根工作区（不支持单文件夹模式）
- 暂存/恢复时相对路径按原样保留
